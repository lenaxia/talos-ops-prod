## Root Cause

From `kubectl describe node worker-00`:

```
Type     Reason     Age   From     Message
Warning  Rebooted    59m   kubelet  Node worker-00 has been rebooted, boot id: 36a23671-9440-4ba5-b242-ee647c3d2380
Normal   NodeReady   59m   kubelet  Node worker-00 status is now: NodeReady
```

Worker-00 experienced a reboot approximately 59 minutes ago. When the node rebooted, the existing node-exporter DaemonSet pod on that node entered a failed/stuck state. The Kubernetes DaemonSet controller detected these failed pods and attempted to kill them (generating the events reported by k8sgpt). The controller then successfully created a new pod (node-exporter-q7wdc), which is now running healthy.

This is a **transient operational event**, not a configuration or code issue. The system behaved correctly and self-healed automatically.

**Note on namespace mismatch:** The k8sgpt finding reports the namespace as "utilities" but the actual node-exporter Service resides in "monitoring". This appears to be a metadata issue in the finding.
